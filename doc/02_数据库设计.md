## 一、表清单总览

### 1.1 系统管理模块（5 张）

| 表名 | 中文名 | 用途 | 关键字段 |
|------|--------|------|---------|
| `sys_user` | 系统用户 | 所有登录账号（员工+业主） | username, password(BCrypt), user_type, community_id |
| `sys_role` | 角色 | RBAC 角色定义 | role_code(唯一), role_name |
| `sys_permission` | 权限 | 菜单/按钮权限（树形） | parent_id, type(1菜单/2按钮), perm_code |
| `sys_user_role` | 用户-角色关联 | 多对多 | user_id, role_id（联合主键） |
| `sys_role_permission` | 角色-权限关联 | 多对多 | role_id, permission_id（联合主键） |

### 1.2 基础数据模块（5 张）

| 表名 | 中文名 | 用途 | 关键字段 |
|------|--------|------|---------|
| `community` | 小区 | 小区基本信息 | name, address, area, build_year |
| `building` | 楼栋 | 归属小区 | community_id, building_no, floors |
| `house` | 房屋 | 归属楼栋 | building_id, house_no, status(1空置/2已售/3已入住/4装修中) |
| `owner` | 业主 | 业主信息 | name, phone, id_card(唯一), plate_no |
| `owner_house` | 业主-房屋关联 | 多对多 | owner_id, house_id, relation, is_primary |

### 1.3 物业服务/业务模块（9 张）

| 表名 | 中文名 | 用途 | 关键字段 |
|------|--------|------|---------|
| `work_order` | 工单 | 业主报修工单 | type, priority, status(1~6), handler_id, rate |
| `work_order_log` | 工单操作日志 | 状态流转记录 | order_id, action, operator_id |
| `fee_item` | 收费项目 | 物业费/水电费/车位费 | name, unit_price, unit, cycle |
| `bill` | 账单 | 应收账单 | house_id, fee_item_id, amount, paid, status(1~4) |
| `payment` | 缴费记录 | 实缴流水 | bill_id, amount, pay_method(1~5) |
| `parking_space` | 车位 | 地上/地下、长租/出售/临时 | space_no, type, owner_id, plate_no |
| `parking_record` | 车位操作记录 | 绑定/续费/解绑流水 | space_id, action |
| `equipment` | 设备 | 门禁/消防/安防三类 | category(1通闸/2消防/3安防), status, next_check_date |
| `equipment_check` | 设备巡检记录 | 巡检结果 | equipment_id, result(1正常/2异常) |

## 二、ER 关系说明

```
community 1───n building 1───n house n───m owner
                                              │
                              owner_house(关联表：relation/is_primary)
                                              │
house 1───n bill n───1 fee_item
              │
              └───n payment

owner 1───n work_order ───n work_order_log
              │
              └───1 sys_user(handler_id 维修员)

owner 1───n parking_space ───n parking_record

equipment 1───n equipment_check

sys_user n───m sys_role n───m sys_permission
        │            │
   sys_user_role  sys_role_permission
```

**核心关系**：
- 小区→楼栋→房屋 三级层级（一对多链式）
- 业主↔房屋 多对多（一个业主可有多套房，一套房可有多个业主成员）
- 账单↔缴费 一对多（一笔账单可多次部分缴纳）
- RBAC 三表关联：用户↔角色↔权限 均为多对多

## 三、公共字段规范

所有业务表均包含以下审计字段：

```sql
id          BIGINT PRIMARY KEY AUTO_INCREMENT  -- 主键
create_time DATETIME DEFAULT CURRENT_TIMESTAMP  -- 创建时间
update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP  -- 更新时间
create_by   VARCHAR(50)                         -- 创建人
update_by   VARCHAR(50)                         -- 更新人
deleted     TINYINT DEFAULT 0                   -- 逻辑删除标记(0未删除/1已删除)
```

## 四、索引设计

| 表 | 索引字段 | 类型 | 用途 |
|----|---------|------|------|
| sys_user | username | 唯一索引 | 登录查询 |
| building | (community_id, building_no) | 唯一索引 | 防止同小区楼栋号重复 |
| owner | id_card | 唯一索引 | 身份证唯一 |
| house | building_id | 普通索引 | 按楼栋查房屋 |
| bill | (house_id, status) | 联合索引 | 按房屋查账单 |
| work_order | (status, handler_id) | 联合索引 | 维修员查待处理工单 |
| parking_space | owner_id | 普通索引 | 按业主查车位 |

## 五、数据字典

### 5.1 用户类型（user_type）
`1` 员工　`2` 业主

### 5.2 房屋状态（house.status）
`1` 空置　`2` 已售　`3` 已入住　`4` 装修中

### 5.3 工单状态（work_order.status）
`1` 待派单　`2` 已派单　`3` 处理中　`4` 已完成　`5` 已关闭　`6` 已撤销

### 5.4 账单状态（bill.status）
`1` 未缴　`2` 部分缴纳　`3` 已缴清　`4` 已作废

### 5.5 支付方式（payment.pay_method）
`1` 现金　`2` 微信　`3` 支付宝　`4` 银行转账　`5` POS

### 5.6 设备类别（equipment.category）
`1` 通闸（门禁闸机）　`2` 消防　`3` 安防（监控）

> 完整数据字典定义在代码 `common/Dict.java`（后端）与 `utils/dict.js`（前端），前后端共享。
