## 功能模块总览（来源：功能规划表）

| 一级模块 | 二级功能 | 分阶段开发要求 |
| -------- | -------- | -------------- |
| 用户认证模块 | 登录、注销、Token 全局鉴权 | 手写 JWT 工具类、拦截器、登录接口 |
| 系统管理模块 | 用户管理、角色管理、权限管理 | 手工编写 RBAC 全套 CRUD、权限控制逻辑 |
| 基础数据模块 | 小区管理、楼栋管理、房屋管理、业主信息管理 | 手写全部接口、分页、导出、前端页面代码 |
| 业务功能模块 | 工单管理、费用管理、车位管理、通闸 / 消防 / 安防设备管理 | 手工完成所有业务逻辑、状态流转、费用计算代码 |
| 数据统计模块 | 仪表盘、收费报表、工单报表、报表导出 | 手写统计 SQL、图表数据接口、报表导出 |

## 接口文档使用约定

- 所有接口 base path：`/api`
- 鉴权方式：请求头携带 `Authorization: Bearer <token>`，详见 [04-接口规范文档](./04-接口规范文档.md)
- 统一返回 `Result<T>` 结构：`{ code, message, data }`
- 时间字段统一为 `yyyy-MM-dd HH:mm:ss` 字符串

## 阅读建议

1. 第一次看项目：先读 **01 项目说明文档** 和 **02 系统架构设计文档**
2. 对接前后端：先读 **04 接口规范文档**，再按模块读 **API-01 ~ API-05**
3. 建库：直接执行 **03 数据库设计文档** 末尾的建库 SQL
4. 跑起来：照着 **05 部署运行文档** 一步步操作
