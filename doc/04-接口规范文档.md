# 04 - 接口规范文档

本文档定义所有接口的**通用约定**，是阅读 API-01 ~ API-05 各模块接口文档的前置基础。每个接口的具体定义在各模块文档中，均遵循本规范。

## 1. 基础信息

| 项 | 值 |
| ---- | ---- |
| Base URL | `/api` |
| 协议 | HTTP / HTTPS |
| 请求编码 | UTF-8 |
| 响应编码 | UTF-8 |
| 数据格式 | `application/json`（文件上传为 `multipart/form-data`） |
| 时间格式 | `yyyy-MM-dd HH:mm:ss` 字符串（日期 `yyyy-MM-dd`） |

## 2. 鉴权

### 2.1 鉴权方式

除登录等白名单外，**所有接口必须携带请求头**：

```
Authorization: Bearer <token>
```

`<token>` 为登录接口（`POST /api/auth/login`）返回的 JWT。

### 2.2 鉴权失败响应

未携带 / 无效 / 过期 token，返回：

```json
{ "code": 401, "message": "未登录或登录已失效，请重新登录", "data": null }
```

### 2.3 白名单接口

| 接口 | 说明 |
| ---- | ---- |
| `POST /api/auth/login` | 登录 |
| `POST /api/auth/refresh` | 刷新 token |
| `GET /api/captcha` | 验证码（如启用） |

## 3. 统一响应结构

所有接口统一返回 `Result<T>`：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { /* 业务数据，可为对象/数组/null */ }
}
```

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| code | int | 业务状态码，200 成功，其它失败 |
| message | string | 提示信息 |
| data | any | 业务数据 |

## 4. 状态码与错误码

| code | 含义 | 触发场景 |
| ---- | ---- | ---- |
| 200 | 成功 | 正常业务 |
| 400 | 参数错误 | 参数校验失败 / 格式错 |
| 401 | 未认证 | 未登录 / token 失效 |
| 403 | 无权限 | 已登录但无该接口权限 |
| 404 | 资源不存在 | 查不到记录 |
| 409 | 业务冲突 | 唯一约束冲突、状态不允许操作 |
| 500 | 服务器错误 | 未捕获异常 |

业务细分错误码（在 2xxx 段，便于前端精细处理）：

| code | 含义 |
| ---- | ---- |
| 2001 | 用户名或密码错误 |
| 2002 | 账号已禁用 |
| 2003 | 旧密码错误 |
| 2004 | 用户名已存在 |
| 2101 | 工单状态流转非法 |
| 2102 | 工单已被接单 |
| 2201 | 账单已缴清 |
| 2202 | 缴费金额超过应收 |
| 2203 | 账单已作废 |
| 2301 | 车位已被占用 |
| 2401 | 设备编号已存在 |
| 9999 | 未知系统错误 |

## 5. 分页约定

### 5.1 请求参数（Query）

| 参数 | 类型 | 必填 | 默认 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| page | int | 否 | 1 | 页码，从 1 开始 |
| size | int | 否 | 10 | 每页条数（最大 100） |

各模块列表接口还支持业务筛选字段，详见各模块文档。

### 5.2 分页响应结构

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "page": 1,
    "size": 10,
    "total": 156,
    "pages": 16,
    "list": [ /* 当前页数据 */ ]
  }
}
```

`PageResult<T>` 字段：

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| page | int | 当前页 |
| size | int | 每页条数 |
| total | long | 总记录数 |
| pages | int | 总页数 |
| list | array | 当前页数据列表 |

## 6. HTTP 方法语义

| 方法 | 用途 | 示例 |
| ---- | ---- | ---- |
| GET | 查询（列表/详情/统计） | `GET /api/system/users/1` |
| POST | 新增 / 业务动作 | `POST /api/system/users` |
| PUT | 全量更新 | `PUT /api/system/users/1` |
| DELETE | 删除（逻辑删） | `DELETE /api/system/users/1` |

> 业务动作（如派单、缴费）用 POST，URL 用动词：`POST /api/business/workorders/1/assign`。

## 7. 路径命名规范

- 全小写，单词连字符，资源用名词复数
- ID 路径参数：`/api/business/workorders/{id}`
- 子资源：`/api/basedata/buildings/{id}/houses`
- 动作：`/api/business/workorders/{id}/{action}`

## 8. 接口文档统一格式

每个接口在模块文档中按以下格式描述：

```
### 接口名
- 请求方法 + 路径
- 权限码（@RequirePermission）
- 简介
- 请求参数（Query / Path / Body，含字段表）
- 请求示例
- 响应示例
- 错误码
```
