# 05 - 部署运行文档

本文档说明如何把「小区物业管理系统」从零跑起来，包含环境准备、数据库初始化、前后端启动、打包与生产部署。

## 1. 环境准备

### 1.1 版本要求

| 软件 | 版本 | 验证命令 |
| ---- | ---- | ---- |
| JDK | 8 或 17（推荐 8） | `java -version` |
| Maven | 3.6+ | `mvn -version` |
| MySQL | 8.0+ | `mysql --version` |
| Node.js | 16+（推荐 18 LTS） | `node -v` |
| npm | 随 Node | `npm -v` |
| Git | 任意 | `git --version` |

### 1.2 推荐开发工具

- 后端：IntelliJ IDEA（含 Maven 插件）
- 前端：VS Code + Volar 插件
- 数据库：Navicat / DataGrip / DBeaver
- 接口调试：Postman / Apifox

## 2. 数据库初始化（必做）

1. 安装并启动 MySQL 8。
2. 执行建库脚本。脚本位于 [03 数据库设计文档](./03-数据库设计文档.md) 的「附录：完整建库 SQL」，可复制保存为 `db/init_property.sql` 后运行。

命令行方式：

```bash
mysql -u root -p < db/init_property.sql
```

或用 Navicat 打开 SQL 文件全选执行。

3. 验证：执行 `SELECT COUNT(*) FROM community;` 应返回非 0；默认管理员账号 `admin` / `admin123` 可登录。

> 原始 Demo 库（`demo` + `user` 表）仍是可用的最小示例，对应 `db/init.sql`，用于验证基础全链路。

## 3. 后端启动（SpringBoot）

### 3.1 修改数据库配置

打开 `backend/src/main/resources/application.yml`，把数据库连接信息改成你自己的：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/property?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的MySQL密码      # ← 必改
    driver-class-name: com.mysql.cj.jdbc.Driver
```

> 注意：库名从 `demo` 改为 `property`（与 03 文档建库脚本一致）。

并补充 JWT 配置（鉴权需要）：

```yaml
jwt:
  secret: YourSuperSecretKeyForJwtAtLeast256bitsLength!!
  expire: 86400
  header: Authorization
```

### 3.2 方式 A：IDEA 启动（推荐）

1. IDEA → Open → 选择 `backend` 目录
2. 等待 Maven 自动下载依赖（首次约 1-2 分钟）
3. 运行 `src/main/java/com/demo/Application.java` 的 `main` 方法
4. 控制台出现 `Tomcat started on port(s): 8080` 即成功

### 3.3 方式 B：命令行启动

```bash
cd backend
mvn spring-boot:run
```

### 3.4 验证后端

浏览器访问 `http://localhost:8080/api/auth/login`（GET 会 405，属正常，说明服务已起）。
或用 Postman POST 登录接口：

```json
POST http://localhost:8080/api/auth/login
{ "username": "admin", "password": "admin123" }
```

返回 token 即正常。

## 4. 前端启动（Vue3）

```bash
cd frontend
npm install        # 首次安装依赖，约 1-2 分钟
npm run dev        # 启动开发服务器
```

看到 `Local: http://localhost:5173/` 后，浏览器打开该地址。

前端 `/api` 请求会被 Vite 代理转发到 `http://localhost:8080`（见 `frontend/vite.config.js`），无需担心跨域。

### 4.1 跨域说明

- 开发环境：Vite proxy 已处理跨域
- 若需直连后端，后端 Controller 上保留 `@CrossOrigin` 兜底

## 5. 验证全链路

| 检查项 | 预期 |
| ---- | ---- |
| 后端控制台 | Tomcat started on 8080 |
| `POST /api/auth/login` | 返回 200 + token |
| 前端 `http://localhost:5173` | 登录页正常 |
| 登录后 | 进入仪表盘，能看到 mock/真实数据 |

## 6. 打包与部署

### 6.1 后端打包

```bash
cd backend
mvn clean package -DskipTests
```

产物：`backend/target/demo-backend-1.0.0.jar`

部署运行：

```bash
java -jar demo-backend-1.0.0.jar \
  --spring.datasource.password=生产数据库密码 \
  --jwt.secret=生产密钥
```

或用 `nohup` 后台运行：

```bash
nohup java -jar demo-backend-1.0.0.jar > app.log 2>&1 &
```

### 6.2 前端打包

```bash
cd frontend
npm run build
```

产物：`frontend/dist/`（静态文件）。

### 6.3 Nginx 部署（生产推荐）

将 `dist/` 上传到服务器，用 Nginx 托管静态资源并反向代理 `/api`：

```nginx
server {
    listen       80;
    server_name  your-domain.com;

    # 前端静态资源
    location / {
        root   /usr/share/nginx/html/property;
        index  index.html;
        try_files $uri $uri/ /index.html;   # SPA history 路由回退
    }

    # 后端接口反代
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 7. systemd 服务（可选，Linux）

把后端注册为系统服务，开机自启。

`/etc/systemd/system/property.service`：

```ini
[Unit]
Description=Property Management Backend
After=network.target mysql.service

[Service]
User=www
WorkingDirectory=/opt/property
ExecStart=/usr/bin/java -jar /opt/property/demo-backend-1.0.0.jar
SuccessExitStatus=143
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable --now property
```

## 8. 常见问题

| 问题 | 排查 |
| ---- | ---- |
| 后端启动报 `Access denied for user 'root'` | `application.yml` 密码不对 |
| 前端请求 401 | token 过期或未登录，重新登录 |
| 前端请求 502 / 网络错误 | 后端未启动，或代理目标端口不对 |
| 前端表单提交 415 | Content-Type 应为 `application/json` |
| 中文乱码 | URL 加 `characterEncoding=utf-8`；数据库字符集 utf8mb4 |
| `npm install` 慢 | 切淘宝源：`npm config set registry https://registry.npmmirror.com` |
| Maven 下依赖慢 | 配置阿里云镜像（settings.xml） |
| 端口被占用 8080 | 改 `application.yml` 的 `server.port`，并同步 `vite.config.js` 代理 target |
