# 02 - 系统架构设计文档

## 1. 总体架构

系统采用 **前后端分离 + 单体应用** 架构。前端为 SPA，通过 HTTP 调用后端 RESTful 接口；后端单体部署，内部分层。

```
┌──────────────────────────────────────────────────────────┐
│                       浏览器 (用户)                          │
└────────────────────────────┬─────────────────────────────┘
                             │ HTTP
┌────────────────────────────▼─────────────────────────────┐
│            前端  Vue3 + Vite + Element Plus                  │
│   路由 / 鉴权拦截 / axios 封装 / Pinia 状态 / ECharts 图表      │
└────────────────────────────┬─────────────────────────────┘
                             │ /api/**  (Vite 开发代理 / Nginx 生产转发)
┌────────────────────────────▼─────────────────────────────┐
│            后端  SpringBoot 2.7 (端口 8080)                  │
│  ┌────────────────────────────────────────────────────┐  │
│  │  Controller 控制层   参数校验 / 调用 Service          │  │
│  ├────────────────────────────────────────────────────┤  │
│  │  Service 业务层      业务逻辑 / 事务                  │  │
│  ├────────────────────────────────────────────────────┤  │
│  │  Mapper 数据访问层    MyBatis SQL                    │  │
│  └────────────────────────────────────────────────────┘  │
│  横切：JwtInterceptor 鉴权 / GlobalExceptionHandler 异常 /    │
│        JwtUtils / BaseEntity 审计字段                        │
└────────────────────────────┬─────────────────────────────┘
                             │ JDBC
┌────────────────────────────▼─────────────────────────────┐
│                     MySQL 8                                │
└──────────────────────────────────────────────────────────┘
```

## 2. 后端分层设计

严格遵循 **四层架构**，职责清晰、单向依赖：

| 层 | 包 | 职责 | 规则 |
| ---- | ---- | ---- | ---- |
| 表现层 | `controller` | 接收请求、参数校验、调用 Service、组装 Result | **不写业务逻辑，不直接调 Mapper** |
| 业务层 | `service` / `service.impl` | 业务逻辑、事务控制 | 可调 Mapper，不返回 Entity 直接暴露 |
| 数据访问层 | `mapper` | 执行 SQL，返回 Entity | 不含业务判断 |
| 实体层 | `entity` / `dto` / `vo` | 数据载体 | Entity 对应表，DTO 入参，VO 出参 |

辅助包：

| 包 | 说明 |
| ---- | ---- |
| `common` | `Result` 统一响应、`ResultCode` 错误码、`PageResult` 分页结果 |
| `config` | `WebMvcConfig`（注册拦截器）、`MybatisConfig`、`CorsConfig` |
| `interceptor` | `JwtInterceptor` 鉴权拦截器 |
| `utils` | `JwtUtils`、`PasswordUtils`(BCrypt)、`UserContext`(ThreadLocal) |
| `exception` | `BusinessException`、`GlobalExceptionHandler` |

## 5. 请求处理流程

以「分页查询用户列表」为例：

```
浏览器
  │  GET /api/system/users?page=1&size=10&username=张
  ▼
Vite 代理 ── 转发 ──▶ http://localhost:8080/api/system/users
  ▼
JwtInterceptor
  ├─ 取 Authorization 头 → 解析 JWT → 校验有效期
  ├─ 失败 → 直接返回 Result(401, "未登录或token失效")
  └─ 成功 → UserContext.set(currentUser)  → 放行
  ▼
SysUserController.page(query)
  ├─ @Valid 校验 query
  ├─ 调 sysUserService.page(query)
  ▼
SysUserServiceImpl.page(query)
  ├─ 业务校验（如数据权限：只能看本小区）
  ├─ 调 sysUserMapper.selectPage(query)
  ▼
SysUserMapper.selectPage  →  SQL  →  MySQL
  ▲
  └─ 返回 List<UserVO> + total
  ▲
Service 组装 PageResult
  ▲
Controller 包装 Result.success(pageResult)
  ▲
浏览器 axios 响应拦截器
  ├─ res.code === 200 → 返回 data
  └─ res.code !== 200 → ElMessage.error + reject
```

## 6. 鉴权方案

### 6.1 登录流程

1. 前端 `POST /api/auth/login` 提交账号密码
2. 后端校验密码（BCrypt 比对），生成 JWT（payload 含 `userId`、`username`、`roles`），有效期 24h
3. 返回 `{ token, userInfo }`
4. 前端把 token 存 localStorage，Pinia 存 userInfo

### 6.2 请求鉴权

- axios 请求拦截器：自动加 `Authorization: Bearer <token>`
- 后端 `JwtInterceptor`：解析 token → 校验 → 取出用户放进 `UserContext`（ThreadLocal）；失败返回 401
- Controller / Service 通过 `UserContext.get()` 拿当前用户

### 6.3 接口白名单（不需鉴权）

```
/api/auth/login        登录
/api/auth/captcha      验证码（如启用）
/api/auth/refresh      刷新 token
```

### 6.4 权限控制

- **接口级**：Controller 方法上加自定义注解 `@RequirePermission("system:user:add")`，拦截器二次校验
- **前端级**：登录后拿权限码列表，动态注册路由 + `v-permission` 指令控制按钮显隐

## 7. 跨域方案

- **开发环境**：Vite `server.proxy` 把 `/api` 转发到 `localhost:8080`，浏览器同源无跨域
- **生产环境**：Nginx 反向代理，前端与接口同域；或后端 `@CrossOrigin` / 全局 `CorsConfig`
- 代码中 Controller 默认加 `@CrossOrigin` 作为兜底

## 8. 异常处理与统一响应

- 所有异常由 `@RestControllerAdvice` 的 `GlobalExceptionHandler` 统一捕获
- 业务异常抛 `BusinessException(ResultCode.XXX)`，返回对应 code/message
- 参数校验异常 `MethodArgumentNotValidException` → 400 + 字段错误
- 未捕获异常 → 500 + "系统繁忙"
- 所有接口统一返回 `Result<T>`，详见 [04 接口规范文档](./04-接口规范文档.md)

## 9. 事务与审计

- `@Transactional` 加在 Service 方法上，`rollbackFor = Exception.class`
- 审计字段（`create_time`、`update_time`、`create_by`、`update_by`）由 MyBatis 拦截器或 AOP 自动填充
- 删除采用**逻辑删除**（`deleted` 字段，0 未删 1 已删），不物理删除

## 10. 配置管理

`application.yml` 关键配置：

| 配置项 | 当前值 | 说明 |
| ---- | ---- | ---- |
| `server.port` | 8080 | 后端端口 |
| `spring.datasource.url` | jdbc:mysql://localhost:3306/demo | 数据库 |
| `spring.datasource.username` | root | 按实际改 |
| `spring.datasource.password` | （改成自己的） | 按实际改 |
| `mybatis.configuration.log-impl` | StdOutImpl | 控制台打印 SQL |

建议新增配置：

```yaml
jwt:
  secret: YourSuperSecretKeyForJwtAtLeast256bitsLength!!
  expire: 86400   # 秒，24 小时
  header: Authorization
```
