## 第一部分：系统安装部署

### 一、环境准备

#### 1.1 版本要求
| 软件 | 版本 | 验证命令 |
|------|------|---------|
| JDK | 8 或 17（推荐 8） | `java -version` |
| Maven | 3.6+ | `mvn -version` |
| MySQL | 8.0+ | `mysql --version` |
| Node.js | 16+（推荐 18 LTS） | `node -v` |
| npm | 随 Node | `npm -v` |
| Git | 任意 | `git --version` |

#### 1.2 推荐开发工具
- 后端：IntelliJ IDEA（含 Maven 插件）
- 前端：VS Code + Volar 插件
- 数据库：Navicat / DataGrip / DBeaver
- 接口调试：Postman / Apifox

### 二、数据库初始化（必做）

1. 安装并启动 MySQL 8。
2. 执行建库脚本（位于 `db/schema.sql`，创建 `property` 库 + 19 张表）：
   ```bash
   mysql -u root -p < db/schema.sql
   ```
3. 执行种子数据脚本（位于 `db/data.sql`，含角色/权限/账号/示例业务数据）：
   ```bash
   mysql -u root -p property < db/data.sql
   ```
   > 用户密码字段在 `data.sql` 中为占位值，由后端 `DataInitializer` 启动时用 BCrypt 加密为正式密码。
4. 验证：执行 `SELECT COUNT(*) FROM community;` 应返回非 0；默认管理员账号 `admin` / `admin123` 可登录。

### 三、启动后端（Spring Boot）

1. 修改 `backend/src/main/resources/application.yml`：
   - 数据库 url：`jdbc:mysql://localhost:3306/property`
   - username / password 改为本机 MySQL 账号
   - 确认 `jwt.secret` / `jwt.expire: 86400` / `jwt.header: Authorization` 配置
2. 启动方式：
   - IDEA：运行 `com.demo.Application` 主类
   - 命令行：`cd backend && mvn spring-boot:run`
3. 成功标志：控制台输出 `Tomcat started on port(s): 8080`
4. 验证：`POST http://localhost:8080/api/auth/login`，body `{"username":"admin","password":"admin123"}` 返回 token。

### 四、启动前端（Vue 3）

```bash
cd frontend
npm install        # 安装依赖
npm run dev        # 启动开发服务器
```
浏览器打开 `http://localhost:5173/`（`/api` 由 Vite proxy 转发到 8080）。

### 五、访问地址
| 项 | 地址 |
|----|------|
| 前端 | `http://localhost:5173` |
| 后端 API | `http://localhost:8080/api` |

### 六、生产部署
- **后端打包**：`mvn clean package -DskipTests` → 产物 `backend/target/property-backend-1.0.0.jar`，`java -jar` 运行。
- **前端打包**：`npm run build` → 产物 `frontend/dist/`。
- **Nginx 部署**：Nginx 托管 `dist/` 静态文件，反向代理 `/api/` 到 `127.0.0.1:8080/api/`；SPA history 路由用 `try_files $uri $uri/ /index.html` 回退。
- 可选：用 systemd 注册为开机自启服务。

## 第二部分：系统使用说明

### 七、系统概述

智慧物业管理系统是一套面向住宅小区物业的 B 端管理平台，帮助物业人员高效完成小区基础信息维护、报修工单处理、费用收取、车位与设备管理、数据统计等工作。本手册面向系统的实际使用者，指导各角色人员完成日常操作。

### 八、登录与注销

#### 8.1 登录
1. 打开浏览器（推荐 Chrome 或 Edge 最新版），访问系统地址。
2. 输入 **账号** 和 **密码**，点击「登录」。
3. 登录成功后自动跳转至数据看板首页。

| 角色 | 默认账号 | 默认密码 |
|------|---------|---------|
| 超级管理员 | admin | admin123 |
| 物业经理 | manager01 | 123456 |
| 收费员 | toll01 | 123456 |
| 维修员 | repair01 | 123456 |
| 业主 | owner01 | 123456 |

> ⚠️ **安全提示**：首次登录后请尽快在「个人中心」修改默认密码。

#### 8.2 修改密码
点击右上角用户头像 →「修改密码」→ 输入旧密码、新密码、确认新密码 →「确定」，修改后需重新登录。

#### 8.3 注销
点击右上角用户头像 →「退出登录」，返回登录页。

### 十、各角色操作指南

#### 10.1 超级管理员
拥有系统全部权限，负责系统配置与用户权限管理。

**用户管理**：
1. 进入「系统管理 → 用户管理」。
2. **新增用户**：点击「新增」，填写用户名、姓名、手机、邮箱、所属小区、用户类型 →「确定」。
3. **分配角色**：点击「分配角色」，勾选对应角色 →「确定」。
4. **重置密码 / 启用禁用**：点击对应操作按钮。

**角色与权限管理**：
1. 进入「系统管理 → 角色管理」。
2. **新增角色**：填写角色编码（如 `TOLL_COLLECTOR`）、角色名称。
3. **配置权限**：点击「分配权限」，在权限树中勾选菜单与按钮 →「确定」。

#### 10.2 物业经理
负责本小区全部业务运营。

**基础数据维护**：
- **小区管理**：「基础数据 → 小区」，维护小区基本信息。
- **楼栋管理**：「基础数据 → 楼栋」，新增楼栋时选择所属小区。
- **房屋管理**：「基础数据 → 房屋」，新增房屋时选择所属楼栋，填写房号、户型、面积、状态。
- **业主管理**：「基础数据 → 业主」，登记业主信息（姓名、电话、身份证、车牌）。
- **业主-房屋绑定**：在业主详情中点击「关联房屋」，选择房屋并标记关系（户主/配偶/子女/租客）。

**报修工单处理**：
- 进入「业务管理 → 报修工单」。
- **派单**：对待派单工单点击「派单」，选择维修员 →「确定」。
- **查看进度**：在工单列表中查看状态（已派单/处理中/已完成）。

#### 10.3 收费员
负责账单生成与费用收取。

**账单生成**：
1. 进入「业务管理 → 账单管理」。
2. 点击「批量生成」，选择收费项目、账期、目标房屋范围 →「确定」，系统自动生成应收账单。

**缴费登记**：
1. 找到待缴费账单（状态：未缴/部分缴纳）。
2. 点击「缴费」，输入实缴金额、选择支付方式（现金/微信/支付宝/银行转账/POS） →「确定」。
3. 全额缴费后状态变为「已缴清」，部分缴费变为「部分缴纳」。

**账单查询与导出**：按状态、账期、房屋筛选；点击「导出」下载 Excel 对账表。

#### 10.4 维修员
负责接收并处理报修工单，执行设备巡检。

**处理工单**：
- 进入「业务管理 → 报修工单」（列表仅显示分配给自己的工单）。
- **接单**：对待处理工单点击「接单」，状态变为「处理中」。
- **完工反馈**：维修完成后点击「完工」，填写处理说明，状态变为「已完成」。

**设备巡检**：
- 进入「业务管理 → 设备管理」。
- **巡检登记**：对待巡检设备点击「巡检」，选择结果（正常/异常），异常时填写描述 →「确定」。
- **到期预警**：临近巡检日期的设备会在列表中高亮提示。

#### 10.5 业主
可查看本人房屋与账单信息，提交报修。

**提交报修**：
1. 进入「业务管理 → 报修工单」。
2. 点击「报修」，选择报修类型（水电/土建/门窗/公共设施/其他），填写问题描述，提交工单。
3. 在列表中查看工单处理进度。
4. 工单完成后可对服务进行评分评价。

### 十一、数据统计使用

**综合数据看板**（登录后默认首页）：
- KPI 卡片：房屋总数、入住率、本月收费、待处理工单数
- 收费趋势图：近 6 个月实收趋势
- 工单状态分布：各状态工单占比
- 设备状态：在线/离线/故障统计
- 待办提醒：待派单工单、到期账单、设备巡检到期

**统计报表**：
- 收费统计：「数据统计 → 收费统计」，查看应收/实收/欠费汇总与趋势。
- 工单统计：「数据统计 → 工单统计」，查看工单按状态、按处理人分布。
- 车位使用率：「数据统计 → 车位统计」。
- 设备状态：「数据统计 → 设备统计」。

## 第三部分：常见问题解答（FAQ）

**Q1：登录提示"账号已禁用"怎么办？**
A：账号被管理员禁用，请联系超级管理员在「用户管理」中启用账号。

**Q2：忘记密码怎么办？**
A：联系超级管理员在「用户管理」中点击「重置密码」，密码将恢复为默认值。

**Q3：为什么我看不到某个菜单？**
A：菜单显示受权限控制。请联系管理员确认你的角色是否分配了对应权限。

**Q4：工单派单后维修员看不到工单？**
A：请确认派单时选择的维修员账号是否正确，且该维修员已登录并具有工单处理权限。

**Q5：账单缴费后状态没有变为"已缴清"？**
A：账单状态根据「应收金额」与「已缴金额」自动计算。部分缴费状态为「部分缴纳」；累计实缴 ≥ 应收时才变为「已缴清」。

**Q6：导出的 Excel 打不开？**
A：请使用 Microsoft Excel 或 WPS 打开；检查浏览器是否拦截了下载。

**Q7：页面数据不刷新？**
A：按 F5 刷新页面，或清除浏览器缓存后重新登录。

**Q8：操作时提示"无权限"？**
A：当前账号无该操作权限。按钮级权限通过 `v-permission` 控制，若可见按钮提示无权限，请联系管理员检查权限配置。

**Q9：后端启动报数据库连接失败？**
A：检查 `application.yml` 中数据库 url、username、password 是否正确，MySQL 服务是否已启动，`property` 库是否已创建。

**Q10：前端启动 `npm install` 很慢或失败？**
A：可切换 npm 镜像：`npm config set registry https://registry.npmmirror.com` 后重试。

## 第四部分：操作注意事项

1. **数据安全**：删除操作不可恢复（逻辑删除），请确认后再执行。
2. **并发操作**：多人同时编辑同一数据时，以最后保存的为准。
3. **浏览器**：推荐 Chrome 或 Edge 最新版，不推荐 IE。
4. **退出习惯**：离开工位时请注销登录，防止账号被盗用。
5. **数据录入**：身份证、手机号等关键字段请确保准确，系统会进行唯一性校验。
6. **密码安全**：默认密码仅用于初始化，正式使用前务必修改；生产环境务必修改 `application.yml` 中的 `jwt.secret`。
