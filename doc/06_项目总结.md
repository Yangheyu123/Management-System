## 二、技术架构回顾

### 2.1 技术栈
| 层 | 技术选型 |
|----|---------|
| 后端框架 | Spring Boot 2.7.18 + Spring MVC |
| 持久层 | MyBatis 2.3.2（注解式 SQL）+ MySQL 8 |
| 鉴权 | JWT（jjwt 0.11.5）+ BCrypt 密码加密 |
| 前端框架 | Vue 3.4（Composition API + `<script setup>`） |
| 前端构建 | Vite 5 |
| UI 库 | Element Plus 2.7 |
| 图表 | ECharts 5.5 |
| 状态管理 | Pinia 2.1 |
| HTTP | axios 1.6 |

### 2.2 架构特点
1. **前后端分离**：前端 SPA + 后端 RESTful API，开发期 Vite proxy 代理。
2. **四层架构**：Controller → Service → Mapper → Entity，职责清晰。
3. **统一规范**：`Result<T>` 统一响应、`GlobalExceptionHandler` 全局异常、`Dict` 状态字典前后端共享。
4. **RBAC 权限体系**：方法级 `@RequirePermission` 注解 + JWT 拦截器 + 前端路由守卫 + `v-permission` 指令，四级权限控制。

## 三、技术难点与解决方案

### 难点一：多级权限控制体系
**问题**：需要实现菜单、按钮、接口、数据四级权限隔离，逻辑复杂。

**解决方案**：
- 后端自定义 `@RequirePermission("system:user:add")` 注解，在 `JwtInterceptor` 中解析当前用户权限码并校验。
- 前端登录后拉取权限码列表存入 Pinia，路由守卫按 `meta.perm` 控制路由访问，`v-permission` 指令按权限码移除无权按钮 DOM。
- 数据级权限通过用户的 `community_id` 限制查询范围。

### 难点二：报修工单全生命周期状态流转
**问题**：工单需经历待派单→已派单→处理中→已完成→已关闭/已撤销多种状态，且每次变更需记录操作日志。

**解决方案**：
- 设计 `work_order` 主表 + `work_order_log` 日志表。
- 在 `WorkOrderService` 中对每个状态变更动作（派单/接单/完工/关闭/撤销/评价）独立封装方法，每个方法内更新主表状态并写入日志。
- 前端工单页面根据当前状态动态显示可用操作按钮。

### 难点三：账单与缴费的部分缴纳处理
**问题**：一笔账单可多次部分缴纳，状态需根据应收与累计实缴自动计算。

**解决方案**：
- `bill` 表记录 `amount`（应收）与 `paid`（累计已缴），`payment` 表记录每笔缴费流水。
- 缴费时在 `BillService` 中累加 `paid`，并按 `paid >= amount` 判断状态（未缴/部分缴纳/已缴清）。
- 使用 `@Transactional` 保证账单更新与缴费流水写入的原子性。

### 难点四：数据统计多维度聚合查询
**问题**：看板与报表需要跨多张表进行聚合统计（收费汇总、工单按状态分布、车位使用率等），SQL 复杂。

**解决方案**：
- 在 `StatMapper` 中编写聚合 SQL，按小区、时间范围等通用维度查询。
- `StatService` 封装各统计方法，前端通过 `EChart.vue` 通用组件渲染折线图、饼图、柱状图。
- 抽取公共查询条件构建方法，减少重复 SQL 拼接（重构后）。

## 六、不足与改进方向

### 6.1 当前不足
1. **未实现业主线上自助支付**：当前缴费由收费员后台登记，未对接微信/支付宝支付网关。
2. **无消息通知**：工单派单、账单逾期等场景缺乏站内消息或短信通知。
3. **无移动端**：仅有 PC Web 端，业主缺少便捷的移动入口。
4. **缓存机制缺失**：统计报表每次实时查询，数据量大时可能影响性能。

### 6.2 改进方向
1. **接入支付网关**：对接微信支付/支付宝，实现业主自助线上缴费。
2. **消息中心**：新增站内消息模块，关键业务节点自动推送通知。
3. **移动端/小程序**：开发业主端微信小程序，支持报修、缴费、查账单。
4. **性能优化**：引入 Redis 缓存统计热点数据，报表查询加索引优化。
5. **数据导出增强**：支持更多维度的报表导出与自定义查询。
6. **部署容器化**：使用 Docker + Docker Compose 一键部署，降低运维成本。
